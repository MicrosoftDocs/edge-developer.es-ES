# <a name="javascript-representation-of-windows-runtime-types"></a>Representación de JavaScript de los tipos de Windows Runtime  

[!INCLUDE [deprecation-note](../includes/legacy-edge-note.md)]  

En la tabla siguiente se describe la forma en que JavaScript representa los tipos de Windows Runtime.  

> [!IMPORTANT]
> Las características de Windows Runtime no están disponibles para las aplicaciones que se ejecutan en Internet Explorer.  

## <a name="windows-runtime-types-in-javascript"></a>Tipos de Windows Runtime en JavaScript  

| Tipo de Windows Runtime | Representación de JavaScript | Descripción |  
|:--- |:--- |:--- |  
| `UInt8` | `Number` | Un Windows Runtime es un entero de 8 bits sin `Uint8` signo, representado como un número de JavaScript.  Un valor de JavaScript primero se convierte en un número de JavaScript y, a continuación, se `ToUint32` sigue el proceso.  Si el valor de número de JavaScript está fuera del intervalo de un Uint8, el resultado tendrá el módulo 2^8 aplicado.  |  
| `Int32` | `Number` | Un Windows Runtime `Int32` es un entero de 32 bits firmado, representado como un número de JavaScript.  Un valor de JavaScript primero se convierte en un número de JavaScript y, a continuación, se `ToInt32` sigue el proceso.  Si el valor de número de JavaScript está fuera del intervalo de un , el resultado tendrá aplicado `Int32` el módulo 2^16.  |  
| `Int64` | `Number` | Un Windows Runtime es un entero de 64 bits firmado, representado como un número estándar si se encuentra dentro del intervalo `Int64` \[-2^53, 2^53\].  Si está fuera de este intervalo, se representa como un valor numérico que tiene un almacén de respaldo personalizado que mantiene los 64 bits completos de datos enteros.  Todas las operaciones matemáticas de estos valores numéricos int64 personalizados hacen que el valor se convierta en un número estándar en el intervalo \[-2^53, 2^53\].  Si el valor está fuera de este intervalo, se genera un error de tipo.  Las excepciones a este proceso de conversión son las operaciones , , y, que comparan los argumentos basados en los 64 bits completos cuando se pasan dos valores representados de `==` `===` `SameValue` `RelationalComparison` 64 bits.  Si cualquiera de los argumentos es un número de JavaScript estándar, estas operaciones también convierten el argumento en un número de JavaScript antes de la comparación.  Un valor de JavaScript se asigna directamente si es un valor int64 en sí; de lo contrario, el resultado de aplicar la `ToInteger` conversión en el valor se pasa a Windows Runtime.  Si se produce un error en la coerción de tipo, se devuelve una excepción.  |  
| `Uint64` | `Number` | Un Windows Runtime es un entero de 64 bits sin signo, representado como un número estándar si se encuentra dentro del intervalo `Uint64` \[0, 2^53\].  Si está fuera de este intervalo, se representa como un número que tiene un almacén de respaldo personalizado que mantiene los 64 bits completos de datos enteros sin signo.  Todas las operaciones matemáticas de estos valores Uint64 personalizados hacen que el valor se convierta en un número estándar en el intervalo \[0, 2^53\].  Si el valor está fuera de este intervalo, se genera un error de tipo.  Las excepciones a este proceso de conversión son las operaciones , , y, que comparan los argumentos basados en los 64 bits completos cuando se pasan dos `==` `===` valores de `SameValue` `RelationalComparison` 64 bits.  Si cualquiera de los argumentos es un número de JavaScript estándar, estas operaciones también convierten el argumento en un número de JavaScript antes de la comparación.  Un valor de JavaScript se asigna directamente si es un valor Uint64 en sí; de lo contrario, el resultado de aplicar la conversión en el valor, seguido de tomar el `ToInteger` módulo 2^52, se pasa a Windows Runtime.  Si se produce un error en la conversión de tipos, se devuelve una excepción.  |  
| `Single` | `Number` | Un Windows Runtime es un número de punto flotante de 32 bits, representado como un número de JavaScript seleccionando el valor de precisión doble más cercano al valor de precisión `Single` única.  Un valor de JavaScript se convierte en un número de JavaScript y, a continuación, se comprueba el intervalo para asegurarse de que el valor se puede representar en un número de punto flotante IEEE 754 de precisión única de 32 bits.  Si se produce un error en la conversión o la comprobación de intervalo, se devuelve un error de cálculo de referencias.  |  
| `Double` | `Number` | Un Windows Runtime `Double` es un número de punto flotante de 64 bits.  `ToNumber` se usa para convertir un Windows Runtime `Double` a un número de JavaScript.  Si se produce un error en la conversión, se devuelve un error de cálculo de referencias.  |  
| `Boolean` | `Boolean` | Un Windows Runtime es un valor de 8 bits donde cero es y cualquier valor distinto `Boolean` de cero es , representado como un `false` `true` JavaScript `Boolean` .  Un valor de JavaScript se convierte primero en un JavaScript `Boolean` mediante `ToBoolean` .  Esto significa que una función de Windows Runtime declarada como podría `void func(BOOL);` escribirse en JavaScript como `obj.func("test");` .  Se llama a la función de Windows Runtime con `BOOL` el parámetro pasado como `TRUE` .  Si se produce un error en la conversión, se devuelve un error de cálculo de referencias.  |  
| `Char16` | `String` | Un Windows Runtime es un carácter Unicode de 16 bits, representado como una cadena `Char16` JavaScript que contiene un solo carácter.  Un valor de JavaScript se convierte en una cadena de JavaScript y, a continuación, se comprueba para asegurarse de que la cadena solo tiene un `ToString` solo carácter de longitud.  A continuación, el carácter único se pasa como `Char16` valor.  Si se produce un error en la comprobación de longitud o conversión, se devuelve un error de cálculo de referencias.  |  
| `String` | `String` | Un Windows Runtime `String` es una secuencia inmutable de `Char16` objetos.  Las cadenas se representan como objetos `String` JavaScript.  Las cadenas de Windows Runtime no tienen valores diferentes para y la cadena `null` vacía \(""\).  `null` y los valores de cadena vacíos se convierten en la cadena vacía de JavaScript.  **NOTA**: Cuando JavaScript se pasa a un parámetro de Windows Runtime que espera una cadena, genera el valor de cadena "null", no o la cadena `null` `null` vacía \(""\).  Las cadenas que se pasan a Windows Runtime siempre deben crearse instancias en la cadena vacía.  |  
| `Enumeration` | `Object` | Un Windows Runtime es un entero de `Enumeration` 32 bits \(signed or unsigned\) que tiene un conjunto asociado de constantes con nombre.  En JavaScript, una enumeración se representa como un objeto que contiene un campo de solo lectura para cada valor con nombre.  Los valores de enumeración se convierten en números de JavaScript tal como se definieron anteriormente para `Int32` y `UInt32` .  Cuando un valor de JavaScript se convierte en una enumeración de Windows Runtime, se convierte, trunca y se comprueba como se describió anteriormente.  Sin embargo, el valor resultante no se comprueba con los valores con nombre definidos que se especifican en la enumeración.  |  
| `Structure` | `Object` | Un Windows Runtime `Structure` es una colección heterogéneo de campos de datos con nombre.  En JavaScript, una estructura se representa como un objeto JavaScript que contiene una propiedad con nombre para cada campo de la estructura.  Si se produce un error en la conversión de cualquier valor de estructura, se devuelve un error de cálculo de referencias.  Se omiten los campos del objeto JavaScript que no tienen equivalentes en la estructura de Windows Runtime.  **NOTA:** No se puede crear una instancia de una estructura de Windows Runtime en JavaScript con la palabra `new` clave.  |  
| `Array` | `Array` | Windows Runtime `Array` se convierte en una matriz de JavaScript.  Sin embargo, las matrices de Windows Runtime no son redimensionables, por lo que algunas operaciones de matriz de JavaScript no son posibles.  La propiedad interna de una matriz convertida no está establecida en "Array" porque la especificación ECMAScript 5 no permite `[[Class]]` esto.  Esto significa que `Array.isArray(v)` devuelve `false` .  Un valor de JavaScript se convierte en una matriz de Windows Runtime de la siguiente manera: Si el valor es o , se convierte `null` en un archivo nativo `undefined` `null` .  Si su propiedad interna es "Array", se copia en una matriz nativa y se pasa una referencia `[[Class]]` a esta matriz.  Si se trata de una matriz convertida, como se describió anteriormente, se pasa la matriz nativa subyacente.  **NOTA:** Si se pasa un valor de matriz de JavaScript a un método de Windows Runtime, se produce una copia completa de la matriz.  |  
| Delegado \(función callback\) | `Function` | Un delegado de Windows Runtime es una referencia a un único método.  Un delegado de Windows Runtime se representa en JavaScript como un JavaScript , que se garantiza que se llamará `Function` en el subproceso adecuado.  Cuando se `Function` invoca, los argumentos se convierten en los tipos de parámetros equivalentes de Windows Runtime.  Si hay menos argumentos de JavaScript que los parámetros `in` delegados, se produce un error en la llamada de delegado.  Si hay más argumentos de JavaScript que parámetros en el delegado, se omiten los `in` argumentos de JavaScript adicionales.  Una vez invocado el delegado, `out` los parámetros se convierten a tipos de JavaScript y se devuelven.  Si un objeto de función de JavaScript nativo se convierte en un delegado de Windows Runtime, se ajusta en un delegado de Windows Runtime del tipo correspondiente.  Cuando se invoca al delegado, los `in` parámetros se convierten en tipos de JavaScript.  Después de invocar al delegado, el valor devuelto se convierte en los `out` parámetros del delegado.  |  
| Interfaz | `Object` | Las interfaces y los grupos de interfaz no se representan directamente en JavaScript como objetos.  Sin embargo, las interfaces se representan como parámetros y devuelven tipos de métodos de Windows Runtime.  Una instancia de interfaz de Windows Runtime se convierte de la siguiente manera:<br /> <br /> 1. Si hay una clase en tiempo de ejecución válida, represente el objeto como una instancia de esa clase.<br /> 2. Si no hay ninguna clase en tiempo de ejecución válida, represente el objeto como una instancia de una clase en tiempo de ejecución sin nombre que implemente exactamente la interfaz que la instancia sabe implementar más sus interfaces necesarias.<br /> <br /> Se prueba un valor de JavaScript para determinar si es una instancia de una clase en tiempo de ejecución o una interfaz.  Si es así, y el valor original de Windows Runtime implementa la interfaz, el objeto se pasa a Windows Runtime.  |  
| Clases en tiempo de ejecución | `Object` | Los objetos de Windows Runtime pueden ser instancias de clases en tiempo de ejecución que implementan una o más interfaces.  Los objetos de Windows Runtime se representan en JavaScript como objetos.  La unión de métodos, propiedades y eventos definidos en todas las interfaces implementadas de la clase representa las propiedades con nombre en el prototipo del objeto JavaScript.  Los consumidores de JavaScript pueden tener acceso a cualquier miembro de la clase directamente.  Los objetos de Windows Runtime tienen un prototipo que contiene todos los miembros de todas las interfaces implementadas de la clase en tiempo de ejecución.  |
