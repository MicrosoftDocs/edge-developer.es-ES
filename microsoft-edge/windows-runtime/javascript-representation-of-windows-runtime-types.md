# Representación de JavaScript de los tipos de Windows Runtime  

En la siguiente tabla se describe la forma en que JavaScript representa los tipos de Windows Runtime.  

> [!IMPORTANT]
> Las características de Windows en tiempo de ejecución no están disponibles para las aplicaciones que se ejecutan en Internet Explorer.  

## Tipos de Windows Runtime en JavaScript  

| Tipo de Windows Runtime | Representación de JavaScript | Descripción |  
|:--- |:--- |:--- |  
| `UInt8` | `Number` | Windows Runtime `Uint8` es un entero de 8 bits sin signo, representado como un número de JavaScript.  Un valor de JavaScript se convierte primero en un número de JavaScript y, a continuación, `ToUint32` se sigue el proceso.  Si el valor de número de JavaScript está fuera del rango de Uint8, el resultado tendrá el módulo 2 ^ 8 aplicado. |  
| `Int32` | `Number` | Windows Runtime `Int32` es un entero de 32 bits con signo, representado como un número de JavaScript.  Un valor de JavaScript se convierte primero en un número de JavaScript y, a continuación, `ToInt32` se sigue el proceso.  Si el valor de número de JavaScript está fuera del intervalo de un `Int32` , el resultado tendrá el módulo 2 ^ 16 aplicado. |  
| `Int64` | `Number` | Windows Runtime `Int64` es un entero de 64 bits con signo, representado como un número estándar si está dentro del rango \ [-2 ^ 53, 2 ^ 53 \].  Si cae fuera de este intervalo, se representa como un valor numérico que tiene un almacén de respaldo personalizado que mantiene todo el 64 bits de datos enteros.  Todas las operaciones matemáticas de estos valores numéricos de Int64 personalizados hacen que el valor se convierta en un número estándar en el intervalo \ [-2 ^ 53, 2 ^ 53 \].  Si el valor está fuera de este intervalo, se genera un error de tipo.  Las excepciones a este proceso de conversión son `==` las `===` operaciones,, `SameValue` y, `RelationalComparison` que comparan los argumentos basándose en los bits de 64 completos cuando se pasan dos valores de 64 bits representados.  Si alguno de los argumentos es un número estándar de JavaScript, estas operaciones también convierten el argumento en un número de JavaScript antes de la comparación.  Un valor de JavaScript se asigna directamente si es un valor Int64; en caso contrario, el resultado de la aplicación de la `ToInteger` conversión en el valor se pasa a Windows Runtime.  Si se produce un error en la coerción de tipos, se devuelve una excepción. |  
| `Uint64` | `Number` | Windows Runtime `Uint64` es un entero de 64 bits sin signo, representado como un número estándar si está dentro del rango \ [0, 2 ^ 53 \].  Si cae fuera de este intervalo, se representa como un número que tiene un almacén de respaldo personalizado que mantiene los 64 completos de datos enteros sin signo.  Todas las operaciones matemáticas de estos valores Uint64 personalizados hacen que el valor se convierta en un número estándar en el intervalo \ [0, 2 ^ 53 \].  Si el valor está fuera de este intervalo, se genera un error de tipo.  Las excepciones a este proceso de conversión son `==` las `===` operaciones,, `SameValue` y, `RelationalComparison` que comparan los argumentos en función de los bits de 64 completos cuando se pasan valores de 2 64 bits.  Si alguno de los argumentos es un número estándar de JavaScript, estas operaciones también convierten el argumento en un número de JavaScript antes de la comparación.  Un valor de JavaScript se asigna directamente si es un valor Uint64; en caso contrario, el resultado de la aplicación de la `ToInteger` conversión en el valor, seguido de módulo 2 ^ 52, se pasa a Windows Runtime.  Si se produce un error en la conversión de tipos, se devuelve una excepción. |  
| `Single` | `Number` | Windows Runtime `Single` es un número de punto flotante de 32 bits, representado como un número de JavaScript, eligiendo el valor de precisión doble más próximo al valor de precisión simple.  Un valor de JavaScript se convierte en un número de JavaScript y, a continuación, se comprueba para garantizar que el valor se puede representar en un número de punto flotante IEEE 754 de 32 de bits de precisión simple.  Si se produce un error en la conversión o la comprobación de intervalo, se devuelve un error de cálculo de referencias. |  
| `Double` | `Number` | Windows Runtime `Double` es un número de punto flotante de 64 bits.  `ToNumber` se usa para convertir un Windows Runtime `Double` en un número de JavaScript.  Si se produce un error en la conversión, se devuelve un error de cálculo de referencias. |  
| `Boolean` | `Boolean` | Windows Runtime `Boolean` es un valor de 8 bits, donde cero es `false` y cualquier valor distinto de cero `true` , se representa como un JavaScript `Boolean` .  Un valor de JavaScript se convierte primero en un JavaScript `Boolean` por `ToBoolean` .  Esto significa que una función de Windows en tiempo de ejecución declarada `void func(BOOL);` podría escribirse en JavaScript como `obj.func("test");` .  La función de Windows en tiempo de ejecución se llama con el `BOOL` parámetro pasado como `TRUE` .  Si se produce un error en la conversión, se devuelve un error de cálculo de referencias. |  
| `Char16` | `String` | Windows Runtime `Char16` es un carácter Unicode de 16 bits, representado como una cadena de JavaScript que contiene un solo carácter.  Un valor de JavaScript se convierte en una cadena de JavaScript `ToString` y, a continuación, se comprueba para garantizar que la cadena solo tenga un carácter de longitud.  Después, el carácter único se pasa como `Char16` valor.  Si la comprobación de la conversión o la longitud no se completa correctamente, se devuelve un error de cálculo de referencias. |  
| `String` | `String` | Windows Runtime `String` es una secuencia inmutable de `Char16` objetos.  Las cadenas se representan como `String` objetos de JavaScript.  Las cadenas de Windows Runtime no tienen valores diferentes para `null` and Empty String \ ("" \).  `null` y los valores de cadena vacíos se convierten en la cadena vacía de JavaScript.  Nota: cuando `null` se pasa JavaScript a un parámetro de Windows Runtime que espera una cadena, genera el valor de cadena "null", no `null` la cadena vacía \ ("" \).  Siempre se debe crear una instancia de las cadenas que se pasan a Windows Runtime a la cadena vacía. |  
| `Enumeration` | `Object` | Windows Runtime `Enumeration` es un entero de 32 bits \ (signed o unsigned \) que tiene un conjunto asociado de constantes con nombre.  En JavaScript, una enumeración se representa como un objeto que contiene un campo de solo lectura para cada valor con nombre.  Los valores de enumeración se convierten en números de JavaScript definidos anteriormente para `Int32` y `UInt32` .  Cuando un valor de JavaScript se convierte en una enumeración de Windows Runtime, se convierte, se trunca y se comprueba el intervalo, según se describe anteriormente.  Sin embargo, el valor resultante no se compara con los valores con nombre definidos que se especifican en la enumeración. |  
| `Structure` | `Object` | Windows Runtime `Structure` es una colección heterogénea de campos de datos con nombre.  En JavaScript, una estructura se representa como un objeto de JavaScript que contiene una propiedad con nombre para cada campo de la estructura.  Si se produce un error en la conversión de cualquier valor de estructura, se devuelve un error de cálculo de referencias.  Los campos en el objeto de JavaScript que no tienen equivalentes en la estructura de Windows Runtime se omiten.  Nota: no se pueden crear instancias de una estructura de Windows Runtime en JavaScript con la `new` palabra clave. |  
| `Array` | `Array` | Windows Runtime `Array` se convierte en una matriz de JavaScript.  Sin embargo, las matrices de Windows Runtime no se pueden cambiar de tamaño, por lo que algunas operaciones de matriz de JavaScript no son posibles.  La `[[Class]]` propiedad Internal de una matriz convertida no se establece en "array" porque esta no está permitida por la especificación de ECMAScript 5.  Esto significa que `Array.isArray(v)` devuelve resultados `false` .  Un valor de JavaScript se convierte en una matriz de Windows Runtime de la siguiente manera: Si el valor es `null` o `undefined` , se convierte en un Native `null` .  Si su `[[Class]]` propiedad Internal es "array", se copia en una matriz nativa y se pasa una referencia a esta matriz.  Si es una matriz convertida, como se ha descrito anteriormente, se pasa la matriz nativa subyacente.  Nota: si se pasa un valor de matriz de JavaScript a un método de Windows Runtime, se produce una copia completa de la matriz. |  
| Delegate \ (devolución de llamada de función \) | `Function` | Un delegado de Windows Runtime es una referencia a un solo método.  Un delegado de Windows Runtime se representa en JavaScript como un JavaScript `Function` , que está garantizado para que se llame en el hilo adecuado.  Cuando `Function` se invoca, los argumentos se convierten en los tipos de parámetro equivalentes de Windows Runtime.  Si hay menos argumentos de JavaScript que parámetros de delegado `in` , se produce un error en la llamada delegada.  Si hay más argumentos de JavaScript que el número `in` de parámetros en el delegado, los argumentos de JavaScript adicionales se pasan por alto.  Una vez invocado el delegado, los `out` parámetros se convierten en tipos de JavaScript y se devuelven.  Si un objeto de función JavaScript nativo se convierte en un delegado de Windows Runtime, se ajusta en un delegado de Windows Runtime del tipo correspondiente.  Cuando se invoca el delegado, los `in` parámetros se convierten en tipos JavaScript.  Una vez invocado el delegado, el valor devuelto se convierte en los `out` parámetros del delegado. |  
| Interfaz | `Object` | Las interfaces y los grupos de interfaz no se representan directamente en JavaScript como objetos.  Sin embargo, las interfaces se representan como tipos de parámetro y devuelto de métodos de Windows Runtime.  Una instancia de la interfaz de Windows Runtime se convierte de la siguiente manera:<br /> <br /> 1. Si hay una clase en tiempo de ejecución válida, represente el objeto como una instancia de esa clase.<br /> 2. Si no hay una clase de Runtime válida, represente el objeto como una instancia de una clase en tiempo de ejecución sin nombre que implementa exactamente la interfaz que se sabe que la instancia implementa más las interfaces requeridas.<br /> <br /> Se prueba un valor de JavaScript para determinar si es una instancia de una clase en tiempo de ejecución o de una interfaz.  Si lo está, y el valor original de Windows en tiempo de ejecución implementa la interfaz, el objeto se pasa a Windows Runtime. |  
| Clases en tiempo de ejecución | `Object` | Los objetos de Windows Runtime pueden ser instancias de clases en tiempo de ejecución que implementan una o varias interfaces.  Los objetos de Windows Runtime se representan en JavaScript como objetos.  La Unión de métodos, propiedades y eventos definidos en todas las interfaces implementadas de la clase representa las propiedades con nombre en el prototipo del objeto de JavaScript.  Los consumidores de JavaScript pueden acceder directamente a cualquier miembro de la clase.  Los objetos de Windows Runtime tienen un prototipo que contiene todos los miembros de todas las interfaces implementadas de la clase en tiempo de ejecución. |
